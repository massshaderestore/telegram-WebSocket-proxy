# TG WS Proxy

**TG WS Proxy** является локальный **MTProto-прокси**, который применяется совместно с **Telegram Desktop**. Для обмена данными с инфраструктурой Telegram применяется транспорт на основе **WebSocket**, за счёт чего соединение передаётся через поддерживаемый канал без изменения логики работы протокола MTProto.

После старта программа прослушивает локальный интерфейс и ожидает соединения клиента **Telegram Desktop**. Когда поступает первое MTProto-соединение, **TG WS Proxy** определяет требуемый **Telegram DC**, устанавливает соединение с соответствующим дата-центром и перенаправляет сетевой трафик. Содержимое MTProto-пакетов не изменяется, поэтому встроенное шифрование Telegram продолжает функционированиеть в штатном режиме.

Как правило, дополнительная инфраструктура не требуется. Для использования достаточно лишь запустить **TG WS Proxy** на локальном компьютере — аренда VPS, конфигурирование промежуточных серверов или использование сторонних MTProto-прокси обычно не нужны.

## Воздопускаетсясти

Ключевые возможности утилиты:

* локальный MTProto-прокси для **Telegram Desktop**;
* передача соединений через **WebSocket**;
* автоматическое определение требуемого **Telegram DC**;
* переключение между **CfProxy** и прямым TCP-подключением если это требуется;
* функционирование в фоновом режиме через системный трей;
* ручная и автоматическая конфигурирование подключения Telegram;
* совместимость Windows, Linux и macOS;
* отдельные сборки для старых версий Windows.

## Схема работы

```text
Telegram Desktop → TG WS Proxy → WebSocket → Telegram DC
```

Во время установки первого соединения утилита открывает локальный MTProto-порт, анализирует начальный пакет MTProto, определяет, к какому **Telegram DC** нужно подключиться, затем инициирует соединение с выбранным дата-центром.

Если установить соединение через **WebSocket** невоздопускается или сервер выполняет перенаправление, **TG WS Proxy** в автоматическом режиме пытается применять **CfProxy**. При отсутствии такой воздопускаетсясти осуществляется прямое TCP-подключение.

## Установка в Windows

Откройте страницу релизов проекта и скачайте сборку, соответствующую используемой версии Windows.

Предлагаются следующие варианты:

* `TgWsProxy_windows.exe` — Windows 10 и новее;
* `TgWsProxy_windows_arm64.exe` — Windows 10 и новее (ARM64);
* `TgWsProxy_windows_7_64bit.exe` — Windows 7 x64;
* `TgWsProxy_windows_7_32bit.exe` — Windows 7 x86.

После старта приложения необходимые параметры для подключения **Telegram Desktop** будут сформированы в автоматическом режиме. Затем **TG WS Proxy** продолжит работу в системном трее, где будут доступны основные функции управления.

## Меню приложения

Основные действия выполняются через контекстное меню значка **TG WS Proxy** в системном трее.

Доступны следующие команды:

* **Открыть в Telegram** — передать ссылку `tg://proxy` непосредственно в Telegram;
* **Скопировать ссылку** — сохранить ссылку подключения в буфер обмена;
* **Перезапустить прокси** — выполнить перестарт локального прокси без закрытия приложения;
* **Настройки...** — открыть окно изменения параметров;
* **Открыть логи** — перейти к журналу работы программы;
* **Выход** — завершить работу **TG WS Proxy**.

## Настройка подключения

### Автоматическое подключение

1. Нажмите правой кнопкой мыши по значку **TG WS Proxy**.
2. Выберите пункт **«Открыть в Telegram»**.
3. Подтвердите добавление MTProto-прокси в **Telegram Desktop**.

Если ссылка не открылась в автоматическом режиме:

1. Используйте пункт **«Скопировать ссылку»**.
2. Отправьте её в раздел «Избранное».
3. Откройте сообщение внутри Telegram.
4. Подтвердите подключение прокси.

### Настройка вручную

Откройте раздел:

**Telegram → Настройки → Продвинутые параметры → Тип подключения → Прокси**

Создайте новое подключение со следующими параметрами:

* **Тип:** `MTProto`
* **Сервер:** `127.0.0.1`
* **Порт:** `1443`
* **Secret:** значение, указанное в конфигурированиех **TG WS Proxy** либо в журнале работы приложения.

Если адрес прослушивания или порт были изменены в конфигурации, используйте актуальные значения вместо стандартных.

## Если возникают проблемы с загрузкой медиа

В отдельных случаях при использовании нескольких записей **DC → IP** загрузка фотографий и видео может функционированиеть нестабильно.

Чтобы проверить, связана ли проблема с конфигурированиеми дата-центров, сделайте следующие действия:

1. Откройте окно настроек **TG WS Proxy**.
2. Найдите раздел **DC → IP**.
3. Удалите все записи, кроме следующей:

```text
4:149.154.167.220
```

Если после этого проблема сохраняется, полностью очистите список адресов **DC → IP**, сохраните изменения и перевыполните старт утилита.

если это требуется также допускается воспользоваться собственным доменом для **Cloudflare Proxy**, выполнив настройку по инструкции из файла **CfProxy.md**.

# Docker

## Использование Docker

**TG WS Proxy** обеспечивает поддержку работу внутри Docker-контейнера, за счёт чего утилита допускается стартать без установки непосредственно в операционную систему. Такой способ особенно удобен для серверов, виртуальных машин и других сред, где применяется контейнеризация.

### Сборка контейнера

```bash
git clone https://github.com/example/tg-ws-proxy.git
cd tg-ws-proxy

docker build -t tg-ws-proxy .
```

### старт

```bash
docker run -d \
  --name tg-ws-proxy \
  --restart=always \
  -p 1443:1443 \
  tg-ws-proxy:latest
```

После старта контейнера ссылку для подключения **Telegram Desktop** допускается получить, просмотрев журнал:

```bash
docker logs tg-ws-proxy
```

Основные параметры контейнера поддерживают настройку через переменные окружения.

| Переменная           | Назначение                        |
| -------------------- | --------------------------------- |
| `TG_WS_PROXY_HOST`   | Адрес прослушивания               |
| `TG_WS_PROXY_PORT`   | Порт приложения                   |
| `TG_WS_PROXY_SECRET` | MTProto Secret                    |
| `TG_WS_PROXY_DC_IPS` | Перечень используемых Telegram DC |

# Конфигурация

Конфигурационные файлы создаются отдельно для каждой поддерживаемой операционной системы.

Расположение каталогов по умолчанию:

* **Windows:** `%APPDATA%/TgWsProxy`
* **Linux:** `~/.config/TgWsProxy`
* **macOS:** `~/Library/Application Support/TgWsProxy`

Пример конфигурационного файла:

```json
{
  "host": "127.0.0.1",
  "port": 1443,
  "secret": "...",
  "verbose": false,
  "pool_size": 4,
  "buf_kb": 256,
  "cfproxy": true,
  "appearance": "auto"
}
```

После изменения параметров сохраните файл конфигурации и сделайте перестарт **TG WS Proxy**. Только после повторного старта утилита начнёт применять обновлённые параметры.

если это требуется также большинство параметров допускается изменить как вручную через конфигурационный файл, так и с использованием соответствующих аргументов командной строки. Это даёт возможность быстро адаптировать работу прокси под различные сценарии использования без изменения исходного кода.

# Настройка Cloudflare Proxy

если это требуется также **TG WS Proxy** может применять **Cloudflare Proxy** для организации подключения к дата-центрам Telegram. Такой вариант полезен в ситуациях, когда прямое соединение функционированиеет нестабильно или недоступно.

## Использование собственного домена

Для максимально стабильной работы желательно применять собственный домен, подключённый к Cloudflare.

Общий порядок параметры:

1. Добавьте домен в аккаунт Cloudflare.
2. Настройте DNS-записи в соответствии с рекомендациями проекта.
3. если это требуется также измените режим SSL/TLS согласно инструкции.
4. Укажите созданный домен в конфигурированиех **TG WS Proxy** вместо стандартного значения.

Использование собственного домена помогает избежать ограничений, которые могут возникать при работе с общими Cloudflare-ресурсами.

## Cloudflare Workers

В качестве альтернативы допускается применять **Cloudflare Workers**. Такой вариант даёт возможность организовать передачу WebSocket-трафика без развёртывания собственного сервера.

Общий порядок параметры выглядит следующим образом:

1. Создайте или войдите в существующий аккаунт Cloudflare.
2. Откройте раздел **Workers & Pages**.
3. Создайте новый Worker.
4. Замените стандартный шаблон кодом, приведённым в документации проекта.
5. Выполните публикацию Worker.
6. Полученный адрес укажите в конфигурированиех **TG WS Proxy**.

После завершения параметры утилита сможет применять опубликованный Worker для передачи трафика к необходимым **Telegram DC**.

# Fake TLS и Nginx

**TG WS Proxy** обеспечивает поддержку работу в режиме **Fake TLS**, что даёт возможность применять прокси совместно с **Nginx** и другими решениями, поддерживающими маршрутизацию соединений по SNI.

При использовании данного режима нужно убедиться, что значение параметра `--fake-tls-domain` соответствует доменному имени, которое указывает на сервер с функционированиеющим прокси.

После параметры Nginx достаточно лишь запустить **TG WS Proxy** с параметрами **Fake TLS**, затем ссылка подключения будет сформирована в автоматическом режиме в формате `tg://proxy`.

# Диагностика

Если во время работы возникают проблемы с подключением, желательно выполнить несколько простых проверок:

* убедиться, что локальный прокси успешно запущен;
* проверить корректность параметров **MTProto**;
* просмотреть журнал работы приложения;
* убедиться в правильности настроек **Telegram DC**;
* проверить доступность используемого **WebSocket**-маршрута;
* если это требуется протестировать работу через **CfProxy** или прямое TCP-подключение.

Во многих случаях этих действий достаточно лишь для определения причины неисправности.
